How does this work?

Where do you find the data?

In order to get information beyond title and abstract (such as number of citations), we are using the database from INSPIRE. The website also provide a convenient API (see here), which allows to directly use the database (or part of it) in Python. The database used on the present website is restricted to articles that were put on arXiv in the category Cosmology and Nongalactic Astrophysics (commonly known as astro-ph.CO).

How do you find recommendations?

The recommendations are based on the similarities in the abstracts. In order to quantify how similar two abstracts are, one needs a vector space and a distance. A powerful way to vectorize sentences is through the use of word embeddings, such as Word2Vec. These type of algorithms allow to represent large corpus in a relatively low dimensional space, while keeping similarities between words: the classical example is that, in the vector space, one typically has Vector(King)-Vector(Man)+Vector(Woman) = Vector(Queen). In the present case, we use the implementation of word2vec in the gensim library for python. More specifically, we use word2vec for the words in the sentences of the abstract. To associate a vector to the full abstract, we take the mean of the word vectors, weighted by the inverse document frequency (IDF), to give more importance to words that are specific to the abstract. Then, we use a K-means clustering algorithm to regroup articles that are similar. The final step is to take the 5 suggestions which are closest to the article, according to their cosine similarity.
